Backend, Callbacks, Promises, and Async Await
Backend= another computer that manages the data of a website
XMLHttpRequest- this is a built-in class provided by the JS
    -creates a new HTTP message to send to the backend, message=request.

Types of requests: 
    GET,POST,PUT,DELETE

URL(Uniform Resource Locator)- Like an address, but for internet
-Helps us locate another computer on the internet.
Ex- https://amazon.com
    https://youtube.com

Request-Response Cycle = 1 request, 1 response
-> A backend only supports a certain set of URL paths

If we send a request to a URL path that is not supported, the backend will respond with an error.
Status Code-
    starts with 4 or 5 (400,404,500) = failed
    4-our problem
    5-backend problem
    Starts with 2(200,201,204) = succeeded

List of URL paths supported is backend API(application programming interface)
inteface=how we interact with something

types of data that backend will respond with are text, html, image
Callback= a function to run in the future

Jasmine has a feature for waiting for some code to finish - done() function
- done() lets us control when to go to the next step

Promises- better way to handle asynchronous code
    -similar to done() function
    -lets us wait for some code to finish, before going to the next step
when we create a promise, it  runs tehe inner function immediately

resolve is a function - similar to done() function
    - lets us control when to go the next step

Promises allow JS to do multiple things at the same time
- lets us flatten our code
use promises instead of callbacks- they keep our code more flat
we can run multiple promises at the same time

Promise.all()-lets us run multiple promises at the same time
    -and waits for all of them to finish

fetch()- better way to make HTTP requests. it uses a promise.

response.json() is asynchronous, it returns a promise

Async Await= even a better way to handle asynchronos code
-> It is a shortcut for promises.
-> async = makes a function return a promise
-> async lets us use await
Await= lets us wait for a promise to finish, before going to next line
-> await lets us write asynchronous code like normal code
** we can only use await, when we're inside an async function
-> the closest function has to be async.
-> Use async await, over promises and callbacks
